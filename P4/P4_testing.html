5. Testing Platforms

Most of our users browsed the page using Chrome on either a desktop or a laptop computer. Some used Mac. We tested the page ourselves using Internet Explorer and Microsoft Edge and spotted no unwanted behaviors.

6. Feeback Summary

Navigation: Users found it very easy to navigate on the page. We recieved no feedback suggesting the oposite.

Content: Sadly, we were unable to test the order page which has a form that is to be sent to our client when someone places an order. Other then that the testers were happy with the look and feel of the galleries.

Design: The testers really liked the images used throughout the page as they felt they really captured the colours and style of ingers products. They also found the page to be really organized.

7. Our Response

Once our client gets a domain name and a host in order, we can test the order page and also add more products to the galleries. We could also expand the page with some sort of news blog showcasing new product series and other announcements. We were happy with how organized the page turned out to be. Most of the images used were taken by a professional photographer, that probably resulted in better pictures than we could have made ourselves.

8. Self-reflection

We found it very difficult to stick to the plan. When we planned the project we had only done a couple of assignments beforehand. This made it kind of hard to try to assess how much work each design decision wold be and how to implement things. The progress plan we made where each person had tasks with deadlines was completely neglected. Througout the course of this project we all learned useful trick on how to simplify certain css tasks and how suprisingly hard some of the seemingly mundane tasks can be. Like alignment with text under pictures that are popping out when you click them, etc... Most important lesson. Stuff takes time.
